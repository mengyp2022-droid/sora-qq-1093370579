# 使用指南

## 这份指南适合谁

这份文档适合第一次接触 `sora-qq-1093370579` 的使用者。它不展开讲所有实现细节，而是帮助你按顺序完成环境准备、服务启动和后台基础配置。

## 第一步：准备运行环境

建议先确认本机已经具备以下基础环境：

- Python 3.10 及以上
- `pip`
- 可用的本地端口 `1989`

如果你准备使用项目中的完整协议能力，除了 Web 后端依赖外，还需要根据实际功能补充安装相关请求库和脚本依赖。

## 第二步：安装基础依赖

在项目根目录执行：

```bash
pip install -r web/backend/requirements.txt
```

这一步会安装 Web 管理后台所需的基础依赖。

## 第三步：启动管理后台

执行下面的命令启动项目：

```bash
python web/run_web.py
```

启动成功后，访问：

```text
http://127.0.0.1:1989
```

默认后台账号通常为：

- 用户名：`admin`
- 密码：`admin123`

首次登录后，建议尽快修改默认密码和关键安全配置。

## 第四步：完成基础配置

进入后台后，建议优先处理“系统设置”中的基础项。常见配置包括：

- 邮箱 API
- 短信 API
- 代理设置
- 打码服务
- OAuth 参数
- 线程数和重试次数

这些配置决定了批量任务、资源池调用和部分自动化流程能否正常运行。

## 第五步：导入资源

项目通常依赖几个资源池共同工作。你可以按下面的顺序准备数据：

1. 导入邮箱资源
2. 导入手机号资源
3. 按需导入银行卡资源
4. 检查账号池是否有可用数据

如果系统设置尚未填好，建议不要急着启动批量任务。

## 第六步：开始日常操作

基础配置完成后，常见操作路径一般如下：

1. 在后台检查系统配置是否保存成功
2. 导入邮箱、手机号等资源
3. 进入批量任务页面启动注册或绑定流程
4. 在账号管理页面查看结果
5. 按需使用视频生成或 Key 管理能力

如果你后续还想继续深入理解项目，建议继续查看这些文件：

## 常见注意事项

- 默认账号密码只适合本地初次启动，不建议长期保留。
- 如果端口被占用，项目可能无法正常启动。
- 如果第三方 API 未配置完整，部分功能页面虽然能打开，但实际任务会失败。
- 使用前请结合相关平台规则、服务条款和当地法律法规自行评估风险。

可以把这份文档理解为“快速上手版”。如果你更关心项目结构，请看根目录下的 `项目介绍.md`；如果你更关心完整配置和背景说明，请继续查看 `README.md` 和 `docs/` 目录。
